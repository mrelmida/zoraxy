Path Policies

Apply an access rule and/or require authentication on specific paths of this endpoint. Rules are evaluated top to bottom and the first matching rule wins.

Order Pattern Regex Access Rule Require Auth Enabled Remove
No Path Policy
Path prefix (e.g. /admin/) or regex (e.g. ^/api/v[0-9]+/.*$).

How path policies work

  • Access Rule: Requests to matching paths are checked against the selected access rule (IP / CIDR / GeoIP white & blacklists) in addition to the host-level access rule. Blocked requests get a 403 response.
  • Require Auth: Requests to matching paths must pass this endpoint's configured authentication provider. If no auth provider is configured, matching requests are rejected with 403.
  • Selective auth: To require authentication only on policy paths (and leave the rest of the site public), enable "Scope authentication to path policies" in the Security tab.
  • Changes are only applied after pressing Save Changes.
